section-1 リファクタリングによる変更 {

    今回の作業によって大幅なリファクタリングが実施された影響で、
    既存のエンティティの実装や役割、およびインターフェースが破壊的に変更されました。
    前回までの命名及びデータモデルはほとんど変更されているため、
    ここに新しい命名及び役割のリストを記載します。

    Model ... torchによるモデルの実装。
        標準的なtorchのインターフェースに従います。

    ModelTrainer ... torchによるモデル学習のヘルパクラス。
        学習はこのクラスのメソッドで行います。(ModelTrainer.train)
    
    ModelTrainer.train_once ... Modelクラスのインスタンスを訓練します。
        1回文のみの逆伝搬実行となるため注意が必要です。

    ModelTrainer.train ... Modelクラスのインスタンスを訓練します。
        maxcountパラメータを設定することで任意の回数の逆伝搬実行が可能。
        実装はModelTrainer.train_onceによって実装されている。

    Sample ... Modelに対するサンプルデータ。
        前回まではテキストとスコアを保持していましたが、
        今回の変更によってBERT出力ベクトルをivecプロパティとして保持し、
        教師出力ベクトルをovecプロパティとして保持します。

    load_samples ... Iterator[Sample]に適合するオブジェクト型のインスタンスを取得します。
        この関数は指定されたパスにある特定の形式に従うcsvファイルから
        BERT出力と教師出力のペアをエントリとするデータセットを取得します。

    save_samples ... Iterator[Sample]に適合するオブジェクト型のインスタンスを保存します。
        この関数は指定されたパスに対して特定の形式従うcsvファイルを
        BERT出力と教師出力のペアをエントリとするデータセットとして保存します。

    load_model ... Modelを取得します。
        任意のパスからModelを読み込みます。

    save_model ... Modelを保存します。
        任意のパスにModelを保存します。
}
