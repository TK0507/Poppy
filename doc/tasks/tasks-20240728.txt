section-1 データ準備 {

    このセクションでは現在の目標であるテキスト分類に沿って、
    どのようなデータを準備するかを検討する。

    まず、準備するデータの形態を検討する。

    少なくとも、テキストとそれに対応する評価値をマッピングしたデータセットが必要です。

    これらを準備するためには、以下のタスクが必要です。

    1.予測結果として得られる属性の決定
    2.テキストデータの準備
    3.テキストデータに対して期待する予測結果

    上記のタスクを実行対象とします。
}

section-2 アプリ作成 {

    このセクションでは現在の目標としてどのようなプログラミングタスクが存在するかを検討します。
    
    テキスト分析モデルであるため、その主な処理はテキストデータおよび評価値のやり取りです。
    そのため、少なくとも以下のようなコンポーネントが必要です。

    > テキストを受け取り、評価値を返却するエントリポイント
    > テキストと期待値から学習を実行するエントリポイント
    > 上記の一般的な制御を提供するインターフェース
    > 上記のユーザー向けフロントエンド

    これらを実装に並び替えると

    1.テキストと期待値から学習を実行するエントリポイント
    2.テキストを受け取り、評価値を返却するエントリポイント
    3.上記の一般的な制御を提供するインターフェース
    4.上記のユーザー向けフロントエンド

    上記のタスクを実行対象とします
}

section-3 まとめ {

    最終的なタスクリストをまとめます。

    下記はデータ準備の項目です。

    1.予測結果として得られる属性の決定
    2.テキストデータの準備
    3.テキストデータに対して期待する予測結果

    下記はアプリ作成の項目です。

    1.テキストと期待値から学習を実行するエントリポイント
    2.テキストを受け取り評価値を返却するエントリポイント
    3.上記の一般的な制御を提供するインターフェース
    4.上記のユーザー向けフロントエンド

    これらを統合する場合、
    予測結果として得られる属性が決定しなければ、
    期待値を設定することもプログラムを作成することもできないため、

    1.予測結果として得られる属性の決定

    は最優先事項です。

    プログラムの作成は、データ準備と並行してある程度実施可能です。
    一応順序を設けるなら、データ準備のほうが優先であるとします。
    そのため、最終的なタスクリストは以下のようになります。

    1.予測結果として得られる属性の決定
    2.テキストデータの準備
    3.テキストデータに対して期待する予測結果
    4.テキストと期待値から学習を実行するエントリポイント
    5.テキストを受け取り評価値を返却するエントリポイント
    6.上記の一般的な制御を提供するインターフェース
    7.上記のユーザー向けフロントエンド
}
