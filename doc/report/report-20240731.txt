section-1 作業結果 {

    今日の作業結果を記録します。

    今日の作業内容は以下の通りです。

    > モデルの多層化
    > Web API化
    > データセットの準備

    モデルの多層化の内容としては、
    1024パラメータを出力する中間層を追加し、Softsign関数を用いております。
    これはReLU系列の関数を回避しつつ、Tanhとは異なる種類の非線形性を導入するためです。
    ReLu系列の関数を回避する理由としては、負の評価値が無効化される可能性が高いからです。
    Softsignの採用に関しては、実際にTanhと比較し、どちらのほうがより精度が改善するかを実験する必要があります。

    Web API化に関しては、flaskによる実装を提供しました。
    REST API形式での実装となり、現在有効なエントリポイントは1つです。
    対象のエントリポイントは/api/beta/predictエントリポイントのみです。
    また、今回の変更によって、
    REST APIに対するJSONスキーマを定義したドキュメントが必要となりました。

    データセットの準備に関しては現在進行中です。
}
